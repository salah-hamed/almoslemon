ملف المشروع المحدث - إصلاح العداد + شريط التقدم + مزامنة محسّنة

تغييرات رئيسية:
1. إصلاح مشكلة الإجمالي: الآن يتم قراءة القيمة من localStorage عند كل نقرة، زيادتها ثم تخزينها — تحل مشكلة عدم ارتفاع الإجمالي.
2. تقدم الهدف اليومي: أضفت شريط تقدم أفقى يعرض النسبة المئوية ونص توضيحي.
3. مزامنة محسّنة: تم تنفيذ طابور محلي (pending_queue) في localStorage. الزيادات تُجمع محليًا ثم تُرسل كـ atomic increments إلى مستند واحد يومي في مجموعة 'daily_totals' كل 60 ثانية أو عند إغلاق الصفحة/الانتقال.
   - هذا يقلل عدد عمليات الكتابة إلى Firestore لأن كل مزامنة تقوم بكتابة دفعة واحدة (document.set مع FieldValue.increment لعدة حقول).
   - ملاحظة عن حدود Firestore المجانية: خطط Spark مجانية تتيح ~20,000 Document Writes/يوم و50,000 Reads/يوم. (المصدر: Firebase docs). لذلك لتجنب تجاوز الحد:
     - نجمّع الزيادات محليًا ونرسل دفعات أقل وعدد مرات كتابة أقل.
     - يمكن التحكم في FLUSH_INTERVAL_MS و MIN_FLUSH_THRESHOLD لتقليل الكتابات.
     - خيار طويل الأمد: استبدال Firestore بتخزين مجمع في BigQuery أو استخدام سيرفر وسيط (proxy) لتقليل الكتابات (يتطلب بنية خلفية).
4. مشاركة محسّنة: تخصيص رسالة المشاركة مع رابط يتضمن معلمات للرجوع (ref، zikr، date). كما أضفت وسوم Open Graph في رأس الصفحة لتحسين الظهور على فيسبوك/واتساب عند مشاركة الرابط (ملاحظة: بعض الشبكات تقوم بجلب المعاينة عند الحملة وتحتاج إلى أن يكون الموقع متاحًا عامًّا على HTTPS).

مصادر سريعة حول حدود Firestore المجانية:
- Firebase Firestore Quotas (وثائق Firebase): Document writes 20,000/day, reads 50,000/day.
